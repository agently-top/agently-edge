## 开发工作流程 (Dev Workflow)

每个 task 按照以下流程执行：

### 1. 脑暴提案 (Brainstorm Proposal)
- 为 task 创建 proposal
- 明确目标、范围、技术方案

### 2. 讨论形成子 plan (Sub-plan)
- 基于提案讨论细化
- 拆解成可执行的任务列表

### 3. Superpowers 执行
- 使用 superpowers 技能的 13 个 integrated skills
- Disciplined AI-assisted development

### 4. TDD 完成开发
- 测试驱动开发
- 先写测试，再实现功能

### 5. 更新状态
- 更新子 plan 状态（完成/进行中/阻塞）
- 用 openspec 更新提案状态

## agently Edge Runtime - 技术架构

**核心栈**:
- 核心运行时：Rust
- LLM 引擎：llama.cpp (C++)
- Linux CLI: Rust (clap)
- Android: Kotlin + JNI (APK)
- 鸿蒙：ArkTS + NDK (HAP)

**交付物**:
- libagently.so (共享库)
- agently CLI (单二进制)
- Android APK
- 鸿蒙 HAP
- Sphinx 用户文档

**开发周期**: 12 周
